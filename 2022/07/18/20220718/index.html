vim

  1. vim中的多行注释和多行删除操作
  • 多行注释
    • 命令行模式下，按Ctrl+v进入列模式
    • 上下选择需要操作的行
    • 按“I”(注意是大写的i)进入插入模式
    • 输入注释符号#
    • 按下ESC，稍等即会出现多行注释
  • 取消多行注释
    • 命令行模式下，按Ctrl+v进入列模式
    • 上下选择需要操作的行
    • 按下x或者d
  • 多行删除操作：仅需命令行模式下，仅需输入“起始行数，结束行数d”即可删除多行
  1. 当md文件没保存，直接关闭下，文件夹下会生成swp文件，再次打开该文件，开始界面会有选择是否看旧文件
  2. 在vimrc文件下设置好autocmd函数，但是，如果直接touch name.md，开头并不会自己写，在vim下:e name.md才会调用SetTitle函数。
  3. :e file关闭当前编辑的文件，开启新的文件，或直接开启新的文件；:saveas newfilename另存为。

code

####代码随想录

  1. 四数相加2
  • 哈希表,解法
    • 该题和两数之和写法类似，两个数组两个数组做
    • map.find(n)查询map中n是否出现
    • map.find(target-a)!=map.end()遍历map，查询是否有target-a的key
  1. 三数之和
  • 双指针解法优于哈希表，因为哈希需要额外的边界条件；
  • 使用双指针，首先需要对给定数组进行排序，sort(nums.begin(),nums.end());sort函数近似快排，此处需要熟悉快排的写法；
  • 在循环过程中，主要去重，这是双指针解法的唯一难点
    • 首先左点需要去重,用来往左边推进
    • 其次三数之和大于小于target(0)都需要去重，都是往内部推进，并且内部已经迭代过，即已经获得更小的r和更大的l
    • 最后等于target也需要去重，此时的去重是为了下一轮的准备，所以需要和r-1,l+1比较
    • 去重后，左右指针均向内移一格，
  1. 四数之和
  • 解法基本和上题一样,注意要排序，否则双指针毫无意义
  • 最外层的两个for下:
    • 去重的逻辑是if(nums[k]==nums[k-1] && k>0) continue;
    • 剪枝的逻辑是if(nums[k]>target && (nums[k]>0 || target>0)) break;
